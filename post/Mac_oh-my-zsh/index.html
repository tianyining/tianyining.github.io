第一步，安装 Homebrew

作为 macOS 必备的包管理工具，相信大家肯定已经很熟悉了，没安装的朋友可以执行下面命令装一下，安装过的可以执行下面命令可以进行更新。

1
2

$ /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

macOS 一般会自带 zsh，不过版本会比较早，我们可以更新一下，以便使用最新特性。自 macOS Catalina（10.15）开始，macOS 系统默认的 Shell 已由 Bash 变为 Zsh。

在终端键入 zsh --version，若显示 Zsh 版本号，则表示 Zsh 已安装。否则需要安装 Zsh。

第三步，切换至 zsh 并安装 oh-my-zsh

切换为 zsh：

1

$ chsh -s /bin/zsh

重启终端即可使用 zsh。

接下来安装 oh-my-zsh

  • 可以通过 curl 或 wget 两种方式来安装，用一条命令即可安装

    curl 安装(失败)

    1

    $ sh -c "$(curl -fsSL https://raw.github.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"

    wget 安装(成功)

    1

    $ sh -c "$(wget https://raw.github.com/robbyrussell/oh-my-zsh/master/tools/install.sh -O -)"

第四步，配置 oh-my-zsh

打开 oh-my-zsh 配置文件

1

$ open ~/.zshrc -a Visual\ Studio\ Code

主题

配置项 ZSH_THEME 即为 oh-my-zsh 的主题配置，oh-my-zsh 的 GitHub Wiki 页面提供了 主题列表 当设置为 ZSH_THEME=random 时，每次打开终端都会使用一种随机的主题。

本次设置主题：

1

$ ZSH_THEME="bira"

插件

1

$ plugins=(git extract osx autojump zsh-autosuggestions zsh-syntax-highlighting)

**注意：**其中 zsh-autosuggestions 和 zsh-syntax-highlighting 是自定义安装的插件，需要用 git 将插件 clone 到指定插件目录下：

1
2
3
4
5

# 自动提示插件
$ git clone git://github.com/zsh-users/zsh-autosuggestions $ZSH_CUSTOM/plugins/zsh-autosuggestions
# 语法高亮插件
$ git clone git://github.com/zsh-users/zsh-syntax-highlighting $ZSH_CUSTOM/plugins/zsh-syntax-highlighting

启用 autojump 插件前，需提前安装 autojump:

1

$ brew install autojump

更新配置

1

$ source ~/.zshrc

遇到的问题及解决

更新配置后，终端报错

1

$ compinit:498: no such file or directory: /usr/local/share/zsh/site-functions/_brew_cask

解决方案：

1

$ brew cleanup
