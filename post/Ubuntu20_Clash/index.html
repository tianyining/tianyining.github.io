问题说明

2022-05-28，Ubuntu20.04 无法使用购买的节点，通过手机端重新连接以及导入更新节点，发现并不是机场跑路或者节点损坏造成的无法科学上网。Qv2ray 2021 年 8 月之前一直有团队维护，是 Ubuntu 下很好用的客户端，但是 2021 年 8 月 后开发者内部矛盾，该项目停止维护，在正常使用将近一年的时间后，目前看 Qv2ray 终究还是挂掉了，逼不得已，需要寻找下一个可在 Ubuntu 下使用的科学上网客户端。

Ubuntu20 下安装使用 clash for windows

目前，Windows Macos 系统下，我均使用 Clash 作为代理客户端，自然 Ubuntu 下也是时候尝试使用 Clash 了。

安装步骤

安装包下载链接： https://github.com/Fndroid/clash_for_windows_pkg/releases

下载 release 界面，如下所示：

将复制的订阅链接粘贴到clash左侧的【配置】选项卡的上方URL处，再点击【下载】，将配置文件导入到Clash中。

之后选择节点，开启科学上网，but fail .

解决方案：手动设置Linux系统代理ip和端口

不知道是不是因为权限的原因，clash 在 linux 上面无法自动打开代理端口，需要手动进行设置，设置方法如下：

首先查看 clash 的默认代理端口是 7890；

打开 Linux 的设置-网络，点击代理后面的设置，将 Network Proxy 设置为 Manual ；照着下图设置 http，https，socks 都设置为 ip 是 127.0.0.1，端口 7890 ，如图所示：

此时理论上可以正常科学上网。
