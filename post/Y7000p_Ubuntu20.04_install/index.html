前言

2021-06-12 线下入手联想拯救者 Y7000p 2020款新机，官方标配以及升级配置如下：

  • **官方标配：**i7-10875H，8 核 16 线程，32G 内存，512G 固态，显存 6G；

  • **升级内寸：**原装机拆箱后发现 16G 内存条是由 2 个 8G 内存条组成，为升级内存至 32G，购买两个 16 G 的内存条（原装对称：内存条的频率必须对应，且内存条容量最好对称）；

  • 升级固态硬盘：2020款的拯救者 Y700p 官方标配是 512G 固态，原装固态是 samsung 512G 固态，预留了一个M.2 SSD的插槽，由于标配的 samsung 1T 固态暂时缺货，最终配置的是 1T SanDisk 固态；

Y7000p 基本信息

查看BIOS模式

win+r快捷键进入cmd ，输入 msinfo32 回车，出现以下界面，可查看BIOS模式：

  1. UEFI模式：

BIOS模式有传统的MBR模式和新式UEFI模式，这将对安装双系统的方法产生直接影响。目前来看，大部分电脑都属于新式UEFI模式，不过也存在一些老机子仍然属于传统MBR模式。本次只介绍新式UEFI模式下的双系统安装方法，如果你的电脑属于传统MBR模式，强烈建议你重装windows系统来更新BIOS模式到UEFI。

  1. 电脑的硬盘数（双硬盘）

Y7000p 官方标配是只有一块 512G 的 SSD，升级之后整机变为 1.5T，512G 是原本的 Windows 系统（磁盘1），升级的 1T 磁盘（磁盘0）用来安装 Ubuntu 系统。

安装前准备工作

一、制作系统盘

使用 UltraISO 制作 Ubuntu 系统安装盘。

二、为 Ubuntu 分配空间⭐️

首先确定电脑是双硬盘。分空间的方法：右键，选择压缩卷。

1. 在Windows系统所在的磁盘，分出525M的空白分区用来安装ubuntu的启动项；

2. 在另外的 1T 空磁盘，在该盘点击右键，选择压缩卷，如下，输入压缩空间量，单位为M；

三、设置BIOS

联想电脑进入 BIOS 的方式是按住开机键后，F2 进入BIOS。

1. 在 Security 页面，关掉 secure boot

2. 进入Boot 页面，将Boot mode [UEFI] 修改为 legacy support

3. Security页面下不仅有Secure Boot选项，看下是否有"clear intel PTT Key"选项，如果有进入回车选选择“yes”清除ppt key,否则无法安装系统后无法引导启动，如下图所示。

4. 保存并退出

保存退出，应该是F10键。

正式安装双系统

设置完 BIOS 后，重启电脑的同时，按住F12，进入 boot manager，选择 U盘 启动，这里选择的是Linpus Lite（TOSHIBA TransMemory）。

安装过程中需要注意的细节

1. 安装过程中使用的语言

一定要选择英文安装；

2. Wireless

为提高安装速度，选择暂时不连接 WiFi；

3. installation type⭐️

一定要选择手动安装，这样才能进行手动分区，否则很有可能删掉原本的 Windows 系统。

分区设置⭐️⭐️

首先让我们回忆在安装前的准备工作中，我们是如何为 Ubuntu 分配空间的：

双硬盘电脑，为安装 Ubuntu 双系统，我们事先分了两块空间：

1. Windows系统所在的磁盘，分了 525M ，计划作为Ubuntu系统的安装启动项的空间；

2. 另一块 1T 磁盘，全部压缩卷，计划作为Ubuntu的分区；

这里我们手动设置分区，先进行Ubuntu系统的安装启动项设置，然后再对Ubuntu系统的分区进行设置：

1）/boot : 找到事先分好的 525M free space，点击 “+” 添加引导分区：

大小：525M

新分区的类型：主分区

新分区的位置：空间起始位置

用于：efi

挂载点：/boot

2）/swap：选择另一块 1T 空闲磁盘 free space，点击 “+” 添加交换分区：

大小：32768M（32G）

新分区的类型：；逻辑分区

新分区的位置：空间起始位置

用于：交换空间

3）/：继续选择空闲磁盘的 free space，点击"+“添加 root 分区：

大小：153600M（150G）

新分区的类型：逻辑分区

新分区的位置：空间起始位置

用于：Ext4日志文件系统

挂载点：/

4）/home：继续选择空闲磁盘的 free space，点击“+”号添加 home 分区：

大小：余下所有空间 813837M（795G）

新分区的类型：逻辑分区

新分区的位置：空间起始位置

用于：Ext4日志文件系统

挂载点：/home

选择安装启动器引导的设备

下面的这一步很重要：在分区界面的下方，选择安装启动项的位置，我们刚刚不是创建了525M的efi分区吗，现在你看看这个区前面的编号是多少，比如是/dev/nvme1n1p7,不同的机子会有不同的编号，下拉列表选择这个efi分区编号（这里一定要注意，windows的启动项也是efi文件，大小大概是500M，而我们创建的ubuntu的efi大小是525M，一定要选对），之后点击"Install Now”

图片中的 Device for boot loader installation 的设备选择是正确的，但是该分区设置有问题，应该是用于 efi ，而不是如图所示的 ext4。

安装成功后，重启电脑

安装成功后，点击重启 restart

按照提示拔掉 U盘，然后 Enter 即可进入 Ubuntu 系统。

选择系统

由于我们把 Windows 和 Ubuntu 的启动项也就是 efi 都安装在了同一块磁盘（磁盘1 512G），因此电脑在开机的时候，会自动在 Windows 系统所在的磁盘中搜索启动项以启动系统；我们当然希望 Ubuntu 的启动项也能够被搜索到，所以将 Ubuntu 的启动项和 Windows 的启动项安装在同一块磁盘中，这样开机时会搜索到 Windows 和 Ubuntu 两个启动项，我们可以手动选择进入哪个系统。

当然这里的"启动项需要安装在 Windows 所在的磁盘"并不是绝对的，只是为了开机能够自动搜索到启动项，安装在其他硬盘也可以，只是每次开机都需要进boot manager（F12）才能找到ubuntu启动项，比较麻烦，所以建议按照我的方法来分区。

踩过的坑

本次安装总体较为顺利，但是第一次尝试安装的过程中，显示安装完毕，需要重新启动，重启后直接进入 Windows 系统，按 F12 进入 boot manager 也没有 Ubuntu 系统的选项，无奈只能重新安装。

问题分析

在 U盘 启动安装 Ubuntu 环节，第一次尝试安装选择的是 USB HHD TOSHIBA Transmemory

这一选择带来的后续问题是，在手动设置分区时，无法将事先分出的 525M 分区用于 efi（因为没有这个选项），导致即使Ubuntu成功安装后，重启电脑，无法找到Ubuntu的启动项，即使 F12 进入 boot manager 也无法找到 Ubuntu 的启动项，因为我们在为Ubuntu系统手动配置分区的时候，就没有设置 efi 项。

解决方案

正式安装 Ubuntu 的时候，插入 U盘 ，重启电脑，F12 进入 boot manager，进入 U盘启动界面，选择 Linpus lite(TOSHIBA TransMemory)，这样在后续的手动分区中，就可以将事先分出的 525M free space 选择用于 efi ，为 Ubuntu 系统设置启动项，安装完毕后重启，拔掉 U盘，Enter 后成功进入 Ubuntu 系统。

选择系统

1）直接按开机键，进入系统选择界面，即 boot manager

按照方式一进入 Ubuntu 系统，无法连接 WiFi 。

2）按开机键，出现 LEGION logo 之前按住 F12 同样会进入到 boot manager。（推荐方式）⭐️

这样的方式进入的 Ubuntu 系统可以连接 WiFi。

遗留问题

虽然使用上述两种方法都能成功进入 Ubuntu 系统，但是按照方式一进入 Ubuntu 系统，无法连接WiFi，原因不清。
