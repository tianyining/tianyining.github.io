问题一

我今天使用git push origin master的时候，提示我：

1

$ fatal: unable to access 'https://github.com/tianyining/tianyining.github.io.git/': Failed to connect to github.com port 443: Operation timed out

分析

在谷歌上查了很多办法都没有解决，有的方法是https连接模式改成ssh模式，或者是修改代理，比如：
git config –global http.proxy 192.168.1.1:8083
但是都没有解决我的疑惑，然后我发现了一个问题，就是我根本没有使用代理，所以这样登录方式肯定会有问题。

解决方案1

通过：

1

$ git config --global http.proxy

查询到当前设置了代理，所以我取消这个设置：

1

$ git config --global --unset http.proxy

再查询，已经没有了代理，然后再push，成功了！

解决方案2

1
2
3

## 取消全局代理：
$ git config --global --unset http.proxy
$ git config --global --unset https.proxy

问题二

同样在使用 git push -u origin master 的时候 ，出现以下报错信息：

1
2
3

(PKU_Summer) ╭─edn-tian@Eden-TiandeMacBook-Air ~/yiningblog/public ‹master› 
╰─$ git push -u origin master
fatal: unable to access 'https://github.com/tianyining/tianyining.github.io.git/': LibreSSL SSL_connect: SSL_ERROR_SYSCALL in connection to github.com:443

解决方案

如果我们使用了代理，请尝试以下解决措施：

1

$ git config --global --add remote.origin.proxy ""
