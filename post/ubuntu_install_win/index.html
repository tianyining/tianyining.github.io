Windows10安装Ubuntu双系统（msiGL62）

写在最前面： 本篇博客记录了本人使用微星电脑GL62安装Windows10+Ubuntu双系统的过程，供我个人在未来重装系统时参考。对于其他有装双系统的人员，也许有借鉴意义。

阅读须知：

  1. 先搞清楚自己的电脑类型：本篇博客仅限于UEFI新式bios+双硬盘（SSD固态硬盘+机械硬盘）+独立显卡（Linux安装过程中或重启过程中存在卡死现象） ；
  2. 本文适用于个人电脑，不适用于服务器；
  3. 推荐刚接触Linux的人使用Ubuntu发行版
  4. 注意下载Ubuntu 18.04 LTS版本，意味着该版本将提供长期免费的安全更新维护支持至2023年4月

准备工作

1.准备材料

  • U 盘：容量 2GB 以上，用于制作 U 盘启动盘，因为在制作启动盘时会格式化 U 盘，所以 U 盘内不要包含重要资料。

  • ubuntu 系统安装镜像（本次使用Ubuntu18.04 LTS版本）

  • 刻录软件，推荐软碟通，会提示注册，选择继续试用就好，不过网络上很多人建议选择rufus或其他，这里不做评价（因为我也不懂(⊙_⊙)?

  • 一个已安装 Windows 的电脑：用于制作 U 盘启动盘

2.配置自己的电脑

  • 首先检查自己的电脑是什么引导（BIOS分为legacy引导和uefi引导）；

怎么看自己电脑是MBR还是UEFI：win+r输入msinfo32回车确认，出现以下信息：

  • 在 WIN10系统下创建空白磁盘分区 这一步就是为即将安装的Ubuntu系统划分磁盘空间，本人选择将F盘所有空间压缩卷（F盘本身为空，331G），压缩后会发现多出一块 未分区磁盘（黑色分区），如果选择的压缩大小是 ×××G， 则黑色的的应该是×××G 可用空间。

  • 制作启动盘 不再赘述

  • win10关闭secure boot （1）按下开机键后同时按住F11（MSI电脑是F11进入BIOS，不同电脑自行更改） （2）电脑重启或开机，也就是在出现微星Logo的时候，连续按F11进入Bios，然后按键盘的上下方向键来选择选项，切换到Security选项卡，然后移动方向键到安全启动菜单并回车进入，找到Secure Boot(安全启动)设置项，将原来开启的状态，设置成关闭(Disabled)状态即可；

记忆中好像还有fast boot（同样需要关掉）

正式安装

写在正式安装前： 准备工作完成后，到这一步，理论上我们就可以开始按住Ubuntu双系统了！ 但是别高兴太早，因为我们是独立显卡GTX1060，Ubuntu对NVIDIA的显卡支持存在bug（据说Ubuntu19修复了这个漏洞，安装时不会出现卡顿），所以在本次安装过程中，卡在了选择语言界面（也可能有部分用户卡在install黑色界面）。

解决方案：

  • 安装时，进入到install黑色界面时，解决办法就是在Install Ubuntu的时候先别急着回车，在选中Install Ubuntu的时候按键盘的e键进入到下面的引导界面

在这里按“e”，进入编辑，修改倒数第二行：

1
2

  quite  splash ---（修改前）
 ** quite  splash  nomodeset**（修改后）

修改过后即可开始顺利安装（虽然安装完成后重启过程仍会遇到卡住问题，但是在这里，我们先开始得以开始安装(⊙o⊙)…）

具体安装过程

  • 安装过程中语言应使用英语，不要使用汉语。若安装界面使用汉语，最终安装的系统中会出现两个比较麻烦的问题：
  1. 家目录下的系统文件夹如 Downloads、Desktop 等会以中文表示，这使得在命令行cd变得很麻烦；
  2. 端中某些字符用英文字体，某些字符用中文字体，效果非常差，需要自己修改终端字体方可；
  • 安装过程中尽量不选择连接WiFi，如果网络畅通的话可能也可以考虑，不过会比较慢；

*** 这一步很重要！！！！！** **！！！！！！！一定要选择something else安装！！！！！！！！** **切记，因为我们要手动分区**

  • 具体分区如下： 点击事先压缩卷生成的空闲区域（free space），点“+”进行手动分配；
  1. efi :（主分区+空间起始） 这个就是实现你双系统的原因了，这个就是用启动 ubuntu 的目录，里面会有系统的引导，这个文件其实只有几十兆，但是我们建议将其划分为 512M 以上文件格式为efi，这个分区必不可少，否则后果你懂得！（本次设置1024M）
  2. swap : （逻辑分区+空间起始） 分了16G（一般情况分8G也是够的）
  3. / :（逻辑+空间起始位置；Ext4）这是 linux 也就是 ubuntu 的根目录就一个反斜杠表示,相当于windows的C盘，本次分了100G；
  4. /opt ：（逻辑+空间起始位置；Ext4）个人习惯是将第三方软件都安装在/opt下，所以分了 100G；
  5. /home :（逻辑+空间起始位置；Ext4）这是 ubuntu的其他盘，相当于windows的其他盘，所以为了让我们自己的目录大一点，剩下的全分给它，文件格式为 ext4，剩余全部空间（137G）。

安装完成后系统会提示重新启动，如果没有卡顿问题，重新启动即代表安装成功

ubuntu双系统启动时卡死解决办法

写在前面：

在安装完ubuntu双系统后，第一次启动ubuntu系统时，卡死在启动界面（或者黑屏），这大概都是由于显卡驱动的原因，具体不在这里阐述，通过以下方法能成功解决，据我个人经验，这可能是诸多方法中最简单最容易理解的方法。

解决方法

  1. （如果已经卡死了，则强制关机），然后重新开机；
  2. （在选择系统的黑色界面）选择ubuntu高级选项，回车；
  3. （在出现的两个模式中）选择恢复（recovery）模式，回车；
  4. （在出现的众多选项中）选择grub，回车（你能看到貌似很牛逼的一行行代码跑过）；
  5. （上面的代码跑完之后应该能重新回到众多选项的界面）选择resume，回车进入系统。

    其实每次开机这样都可以进入系统，但是很麻烦，所以下面我们进行永久修改（前提是你已经通过上面的方式进入了系统）：

1. 修改/etc/default/grub文件： 终端输入：

1

$ sudo gedit /etc/default/grub

在打开的文件中，将其中的quiet splash修改为quiet splash nomodeset并保存;

2. 更新修改完的grub 终端输入：

1

$ sudo update-grub

回车即可，以上，问题完美解决，以后进系统正常进就行。

改变分辨率

写在前面： 电脑成功重启后，发现Ubuntu系统显示变形了，是分辨率的问题

解决方法

1. 打开终端，修改开机引导文件：

1

$ sudo gedit /etc/default/grub

2. 找到文件中的GRUB\_GFXMODE，并将其改为GRUB\_GFXMODE=1920x1080（你想要的分辨率），并去掉GRUB\_GFXMODE=1920x1080 前面的‘#’注释号

3. 执行：

1

sudo update-grub

4. reboot（重启）
